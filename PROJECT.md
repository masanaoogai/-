## 1. プロジェクト概要

### アプリ名
**まいにち脳活**（Mainichi Nokatsu）

### コンセプト
高齢者向けの認知機能トレーニング・スクリーニングポータルアプリ。  
毎日3分で脳を元気に。認知機能を楽しく・毎日・続けやすく。

### 形態
- **PWA（Progressive Web App）**
- タブレット横向き（landscape）専用設計
- iPad Safari からホーム画面に追加して全画面起動

### 3-4. ゲームプレイヤー仕様

- ポータル内フルスクリーンiframeでゲームを表示
- ヘッダーバーは起動4秒後に自動非表示
- 画面タップで再表示
- 「もどる」でゲーム進行ページに戻る
- `sandbox` 属性で安全にiframe表示

### 3-5. 認証・ユーザー管理（Supabase）

| 機能 | 内容 |
|---|---|
| ログイン | メールアドレス＋パスワード |
| 新規登録 | 名前＋メール＋パスワード |
| ゲストモード | 登録なしで利用可（結果は保存されない） |
| セッション復元 | ページリロード時に自動ログイン維持 |
| ヘッダー表示 | ログイン後、メニューヘッダー右上にアカウント名＋アバター |
| 結果保存 | ゲーム完了時に `game_results` テーブルへ自動保存 |
| Supabase未設定時 | 自動でゲストモードで起動（エラーにならない） |

### 3-6. ユニバーサルデザイン設定（Settings）

| 設定項目 | 内容 |
|---|---|
| 文字の大きさ | 標準 / 大きい(120%) / とても大きい(145%) |
| 画面の色・見た目 | 通常 / ダークモード / 白黒反転 / グレースケール |
| ボタン枠を強調 | 枠線を5px太く表示 |
| 行間・文字間を広げる | 行間1.9倍・字間0.06em |
| アニメーションを止める | 全トランジション無効化 |
| 音声読み上げ | ボタンタップ時にWeb Speech APIで読み上げ（日本語） |
| 読み上げ速度 | ゆっくり(0.7x) / 普通(1.0x) / はやい(1.4x) |

設定はlocalStorageに自動保存。次回起動時も継続。

### 3-7. デザイン仕様

- 背景色: `#FFFBF0`（クリーム）
- メインカラー: `#22C55E` → `#15803D`（グラデーション緑）
- フォント: Hiragino Kaku Gothic ProN / Hiragino Sans / Meiryo
- タブレット横向き専用（`landscape` 固定）
- 1画面完結（スクロールなし）
- 文字サイズ: 最小18px、ボタン22〜48px（高齢者配慮）
- タップエリア: 最小44px × 44px

## 4. admin.html — 管理画面仕様

### 4-1. 概要
ポータルのゲームURLを管理するCMS。  
同一ドメインにデプロイして使用。設定はlocalStorageで共有。

### 4-2. タブ構成

| タブ | 内容 |
|---|---|
| トレーニングゲーム | 7カテゴリ × 10枠のURL・ゲーム名を管理 |
| 総合テスト | テスト1（4ゲーム）・テスト2（4ゲーム）のURLを管理 |
| 設定 | Supabase接続設定・エクスポート/インポート/リセット |

### 4-3. 機能一覧

| 機能 | 説明 |
|---|---|
| URL入力 | 各ゲームのURLを入力欄に設定 |
| 全て保存 | localStorageに保存→ポータルに自動反映 |
| ▶ 確認 | URLが正しく開くか別タブでテスト |
| ✕ クリア | URLを空にする（準備中に戻す） |
| エクスポート | 全URL設定をJSONファイルとして保存 |
| インポート | JSONファイルから設定を復元 |
| リセット | 全URL設定を初期化 |
| Supabase設定 | Project URL・Anon Keyを設定（ポータルが自動読み込み） |

### 4-4. ゲーム名の更新

空き枠（6〜10番）は管理画面でゲーム名を入力可能。  
名前を入力してURLも設定すると、ポータルに新ゲームとして反映される。

## 6. sw.js — Service Worker仕様

- **Cache First** 戦略（オフライン時もキャッシュから起動）
- キャッシュ対象: `index.html`, `manifest.json`, `icons/*`
- 外部ゲームURL（bolt.host等）はキャッシュしない
- バージョン: `nokatsu-v1`

## 7. Supabase DB 設計

### テーブル構成

#### `profiles` テーブル
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | auth.usersと紐付け |
| display_name | text | 表示名 |
| created_at | timestamptz | 登録日時 |

#### `game_results` テーブル
| カラム | 型 | 説明 |
|---|---|---|
| id | bigserial (PK) | 自動採番 |
| user_id | uuid (FK) | ユーザーID |
| game_name | text | ゲーム名 |
| course_id | text | コースID（総合テスト用） |
| completed | boolean | 完了フラグ |
| duration_sec | integer | プレイ時間（秒） |
| score | integer | スコア（拡張用） |
| memo | text | メモ（拡張用） |
| played_at | timestamptz | プレイ日時 |

### RLS（Row Level Security）
- 自分のデータのみ参照・書き込み可能
- 他ユーザーのデータは一切参照不可

## 10. 開発履歴（チャット作業ログ）

### Phase 1 — bolt.new からの移植
- bolt.newで作成したプロジェクト（ZIP）をClaudeにアップロード
- React + Vite + TypeScript 構成を単一HTMLに変換
- 全ページのナビゲーション・ゲーム選択ロジックを再現

### Phase 2 — デザインリニューアル
- bolt.newのデザインをClaudeで段階的に改善
- トップページ・メニュー・総合テスト・ゲーム進行の各ページをリデザイン
- 高齢者向け：文字大きめ・ボタン大きめ・シンプルなUI

### Phase 3 — PWA 対応
- `manifest.json`（display: fullscreen, orientation: landscape）
- `sw.js`（Service Worker、Cache First戦略）
- iOS/Android 向けメタタグ追加
- PWAインストールバナー実装

### Phase 4 — タブレット横向き最適化
- 全ページを `100vw × 100vh` に収める設計
- スクロールなし・1画面完結のレイアウト

### Phase 5 — 管理画面（admin.html）の作成
- ゲームURLをGUI操作で設定できる管理画面
- localStorageでポータルと設定を共有
- エクスポート/インポート機能
- ゲーム枠を5→10に拡張（空き枠対応）

### Phase 6 — ゲームプレイヤー（フルスクリーンiframe）
- 外部ゲームを別タブではなくポータル内iframeで表示
- 自動バー非表示（4秒後）→ タップで再表示
- manifest.jsonを `standalone` → `fullscreen` に変更

### Phase 7 — 総合テストの一気通貫プレイ
- 総合テスト1・2をコースとして定義
- 選択→確認→スタートで複数ゲームをキュー管理
- 各コースのゲーム構成を仕様化

### Phase 8 — UD（ユニバーサルデザイン）設定
- 設定ページを新設（メニューに追加）
- 文字サイズ・テーマ・枠線強調・行間・アニメーション停止
- Web Speech API による音声読み上げ（日本語対応）
- 設定はlocalStorageに自動保存

### Phase 9 — ログイン・認証機能
- ログインページを新設（起動時に表示）
- Supabase Auth によるメール認証・新規登録
- ゲストモード（Supabase未設定でも動作）
- ヘッダー右上にアカウント名・アバター表示
- ゲーム結果をユーザー別にDBへ保存
- RLSで自分のデータのみ参照可能

### Phase 10 — エラー修正
- scriptブロックが3つに分割されていた問題を修正
- 全JSを1つのscriptブロックに統合
- Supabase CDNをheadタグの静的ロードから動的ロードに変更
  （Claude.aiアーティファクット環境でのCDNブロックに対応）
- `startCourse()`・`launchCourse()` が消失していた問題を修正
- `page-game-player` HTMLが欠落していた問題を修正
- 全関数・全ページの整合性確認済み
